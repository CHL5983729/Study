# 联合索引最左前缀原则

## 1.什么是联合索引

联合索引是数据库表上建立在多个列上的一个索引。

列如：有一个user表，包含name，age，city三列，可以创建一个联合索引(name,age,city)

### 2.为什么使用联合索引

提高多条件查询的效率，实现索引覆盖，避免回表，减少磁盘IO

### 3.什么是最左前缀原则

当使用联合索引时，查询条件必须从最左边开始，且不能跳过中间的列，索引才能被有效利用

### 4.核心规则

假设联合索引为 `(a, b, c)`，以下查询能用上索引：

| 查询条件                        | 是否走索引  | 说明                         |
| --------------------------- | ------ | -------------------------- |
| `a = 1`                     | 全用     | 只用到了 a 列                   |
| `a = 1 AND b = 2`           | 全用     | 用到了 a 和 b                  |
| `a = 1 AND b = 2 AND c = 3` | 全用     | 三列都用                       |
| `a = 1 AND c = 3`           | 但部分失效  | a 列用了索引，c 列无法用（跳过了 b）      |
| `b = 2`                     | 无法使用   | 没有从最左边 a 开始                |
| `b = 2 AND c = 3`           | 无法使用   | 缺少 a                       |
| `a = 1 AND b > 0 AND c = 3` | 但 c 失效 | a 用索引，b 用索引（范围查询），c 在范围后失效 |

### 5.为什么会有这个原则

这是由B+树的数据结构决定的。联合索引的排序是先按第一列排，第一列相同在按第二列排，第二列相同在按第三列排。

如果没有第一列条件，就不知道从树的哪个分支开始查询，如果跳过了中间列，则无法使用二分查找快速定位，怎能扫描所有数据，在筛选

### 6.如何正确使用最左侧原则

1.把最长作为查询条件的列放到左边

2.考虑索引的排序和覆盖

3.不要跳过中间列
