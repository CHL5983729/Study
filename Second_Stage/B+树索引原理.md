# B+树索引原理

## 1.为什么需要索引

1.1 一个数据库，如果想要找一条数据，必须从第一条数据查询到最后一条数据,这叫“全表扫描”，效率是非常低的，而有了索引，就可以快速定位到数据所在的地方

## 2.B+树长什么样

2.1 B+树是一种多路平衡查找树，它由两类节点组成：

（1） 内部节点 ：之存键值（比如学号，主键ID）不存实际数据，作用是“指路”：告诉你该向哪个子节点走，一个内部节点可以有多个子节点

（2）叶子节点：存储键值+指向实际数据的指针，所有子叶节点按键值从小到大排序，子叶节点之间通过指针连接成一个有序列表

# 3.B+树三大特性

（1）多路：每个节点有多个子节点，所以树的高度非常低

（2）平衡：所有叶子节点在同一层，插入/删除后自动调整，保持高度一致

（3）有序：叶子节点通过链表连接，支持高效的范围查询

| 原理是什么？           | 多路平衡查找树，所有数据在叶子，叶子间有链表。  |
| ---------------- | ------------------------ |
| 为什么快？            | 矮胖（高度低）、内部节点扇出大、每次IO读一页。 |
| 范围查询为什么快？        | 叶子链表直接顺序遍历。              |
| 插入/删除怎么办？        | 自动分裂/合并，保持平衡。            |
| 为什么数据库用B+树而不是B树？ | 范围查询效率高，内部节点更节约空间，IO更少。  |
