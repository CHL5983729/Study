# MySQL dump恢复与备份

## 1.什么是MySQL dump

MySQL dump是MySQL自带的逻辑备份工具，能将数据库中的数据和结构导出为SQL脚本

优点：兼容性好，可跨版本，跨平台，支持灵活的备份粒度

缺点：大数据库备份和恢复速度慢，备份期间会锁表

## 2.需要的权限

备份用户需要以下权限：读取数据（select），备份视图（show viem），备份触发器（trigger），备份时锁表（lock tables）

恢复用户需要以下权限：创建数据库和表（create），插入数据（insert），修改表结构（alter），覆盖原有数据时需要（drop）

## 3.备份操作

基本语法：

```sql
mysqldump [选项] 数据库名称 [表1，表2,....] > 备份文件名.sql
```

备份常用选项：

| 选项                                | 说明                          |
| --------------------------------- | --------------------------- |
| `-u 用户名`                          | 指定 MySQL 用户名                |
| `-p`                              | 提示输入密码（不要在 - p 后直接写密码！）     |
| `-h 主机名`                          | 指定 MySQL 服务器地址（远程备份时用）      |
| `-P 端口号`                          | 指定 MySQL 端口（默认 3306）        |
| `--single-transaction`            | **InnoDB 引擎必备**，创建一致性快照，不锁表 |
| `--databases`                     | 备份多个数据库                     |
| `--all-databases`                 | 备份所有数据库（包括 mysql 系统库）       |
| `--no-data`                       | 只备份表结构，不备份数据                |
| `--no-create-info`                | 只备份数据，不备份表结构                |
| `--routines`                      | 备份存储过程和函数                   |
| `--triggers`                      | 备份触发器（默认开启）                 |
| `--events`                        | 备份事件调度器                     |
| `--default-character-set=utf8mb4` | 指定字符集，避免乱码                  |
| `--set-gtid-purged=OFF`           | MySQL 8.0 主从复制环境必备          |

## 4.恢复操作

恢复前需要先创建一个空的数据库

恢复语法：

```sql
mysql -u 用户 -p 数据库名 < 备份文件.sql 
```
