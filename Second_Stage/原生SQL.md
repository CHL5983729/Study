# 原生SQL

## 1.为什么需要使用原生SQL

SQLALchemy ORM 虽然强大，但是在以下场景种原生SQL是更好的选择：

1.复杂查询：多表连查，子查询，窗口函数等ORM难以表达的逻辑

2.性能优化：对性能要求极高的场景，手写SQL比ORM生成更高效

3.数据库的特有功能：PgSQL的JSONB操作，MySQL的全文索引等

4.批量操作：大批量数据插入/更新，ORM效率低下

5.遗留系统：已经有写好的复杂SQL，不会使用ORM重写

## 2.使用方式

session.execute(text(       "SQL语句"      )) 

1.必须使用text()包裹SQL语句

2.绝对不要使用字符串来拼接SQL语句，否则会导致SQL注入
