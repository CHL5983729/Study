# Redis 缓存穿透、击穿、雪崩

这三个问题本质上都是同一个核心问题：大量请求绕过了缓存，直接打到了数据库，把数据库压垮

缓存的基本工作原理：

```text
用户请求        --->       先查Redis缓存 
                              ↓
                         缓存中有数据吗？
                              ↓
                          ┌───┴───┐
                         有      没有
                         ↓        ↓
                      返回数据  查数据库
                                  ↓
                            数据库有数据吗？
                                  ↓
                              ┌───┴───┐
                             有      没有
                             ↓        ↓
                         写入Redis   返回空
                          返回数据
```

## 1.缓存穿透

缓存穿透指的是：用户请求的数据，在缓存中没有，在数据库中也没有，这样的请求每次都会绕过缓存，直接打到数据库，如果有大量这样的恶意请求，数据库就会被压垮

为什么会发生：标准缓存的流程有一个漏洞：如果数据库返回为空，那么就不会写入缓存，所以下次相同的查询还是会查数据库

解决方案：

1.缓存空值：当数据库返回空时，把这个空值缓存起来，设置一个较短的过期时间

2.布隆过滤器：提前把所有存在的数据ID，存入一个布隆过滤器，请求过来时，先查布隆过滤器，如果没有，直接返回空，如果有才继续查询缓存和数据库

3.接口层校验：在请求达到缓存之前，先做参数合法校验

## 2.缓存击穿

缓存击穿指的是：一个非常热点的Key，在它过期的瞬间，有大量的并发请求同时过来，这些请求会同时发现数据库中没有数据，同时去查询数据库，导致数据库瞬间被这一个Key的大量请求压垮

为什么会发生：缓存的过期时间是统一的，如果一个热点Key设置了一小时过期，那么一小时后，所有请求都会打到数据库

解决方案：

1.对于热点Key，直接不设置过期时间，然后用一个后台线程，定时去更新这个Key的缓存

2.互斥锁：当缓存失效时，不是所有请求去查数据库，而是只让第一个请求去查，其他请求等待，等第一个请求把缓存写好后，其他请求直接查缓存

```text
大量请求同时过来 → 查缓存 → 没有
                         ↓
                    尝试加分布式锁
                         ↓
                     ┌───┴───┐
                  加锁成功  加锁失败
                    ↓        ↓
                查数据库   等待100ms
                写入缓存   再查缓存
                释放锁     返回数据
                返回数据
```

3.逻辑过期

原理：不设置真正的过期时间，而是在value里面存一个过期时间，当请求过来时，发现逻辑时间过期了，就异步更新缓存，同时直接返回空数据

## 3.雪崩

雪崩指的是：大量的Key在同一时间过期，或者整个缓存服务器宕机了，会导致：所有请求都绕过缓存，直接打到数据库，数据库瞬间被压垮

为什么会发生：批量设置了相同的过期时间，缓存服务器不可使用

解决方案：

1.在设置过期时间时，给每个Key的过期时间加上一个随机值，这样就不会出现大量Key同时过期的情况

2.服务降级与熔断，当缓存不可使用时，不是让所有请求都去查数据库，而是直接返回降级数据或者熔断接口

3.搭建Redis主从集群+哨兵模式，或者Redis Cluster集群，保证Redis服务不会单点故障

| 问题   | 核心原因                 | 发生场景          | 最典型特征           | 首选解决方案       |
| ---- | -------------------- | ------------- | --------------- | ------------ |
| 缓存穿透 | 请求的数据在缓存和数据库中都不存在    | 恶意攻击、参数错误     | 单个不存在的 ID 被大量请求 | 缓存空值 + 布隆过滤器 |
| 缓存击穿 | 单个热点 key 突然过期        | 秒杀、热门新闻       | 单个存在的 ID 被大量请求  | 热点 key 永不过期  |
| 缓存雪崩 | 大量 key 同时过期 或 缓存服务宕机 | 系统上线、Redis 宕机 | 所有请求都打到数据库      | 过期时间加随机值     |
